## 三、技术方案

### 图片生成

- **文件：** 修改 `generate_images.py`，在 `PROMPTS` 字典末尾追加 25 个事件条目
- **路径格式：** `events/{event_id}`，输出到 `images/events/{event_id}.png`
- **尺寸：** 统一 `1920x1080`（16:9 宽幅）
- **风格：** 复用现有 `STYLE` 前缀（中国水墨画风格）
- **目录：** `main()` 中 `subdirs` 列表追加 `"events"`
- **模型：** 使用当前配置的 `gemini-3-pro-image-preview`
- **断点续传：** 已有逻辑自动跳过已生成的图片

### JSON 配图字段

在 4 个阶段事件 JSON 文件中，给精选的 25 个事件添加 `"image"` 字段：

```json
{
  "id": "youth_01",
  "title": "虎牢关前的少年",
  "image": "images/events/youth_01.png",
  ...
}
```

- 仅给精选的 25 个事件添加，其余事件不加此字段
- `"image"` 字段放在 `"id"` 和 `"title"` 之后，`"description"` 之前

### UI 显示

修改 `js/ui.js`，在事件渲染逻辑中：

1. 检查事件对象是否有 `image` 字段
2. 有图：在事件描述上方插入 `<img>` 元素
   - `width: 100%`，圆角 `border-radius: 8px`
   - `opacity` 淡入动画（0 → 1，0.5s）
   - 图片加载失败时静默隐藏（`onerror` 设 `display:none`）
3. 无图：保持现有纯文字，无变化

## 四、提示词风格规范

所有事件插画提示词遵循统一格式：

```
[场景具体描述，含人物、动作、环境、氛围]，{STYLE}，宽银幕电影画面
```

- 每个提示词 40-80 个中文字符
- 描述具体视觉元素（人物姿态、环境细节、光影氛围）
- 结尾统一加 `宽银幕电影画面` 后缀
