## 1. 游戏概述

「三国武将录」是一款三国题材的 H5 文字冒险游戏，玩法类似修仙模拟器。玩家选择一位历史武将，经历从少年到暮年的人生旅程，在一系列随机事件中做出选择。不同的选择改变武将的属性值，最终属性决定武将的命运结局。

**核心循环：** 选将 → 随机事件 → 做出选择 → 属性变化 → 触发结局

**目标用户：** 喜欢三国题材、轻度策略养成、碎片化游玩的 H5 玩家

### 2.2 属性系统

**四维属性（玩家可见）：**
- **武力（武）**：战斗能力，影响战争类事件结果
- **智谋（智）**：计策能力，影响谋略类事件和隐藏选项
- **品德（德）**：道德品行，影响人际关系和忠义类事件
- **魅力（魅）**：领袖气质，影响招募、结盟类事件

属性范围：0-100

**隐藏属性（不直接显示给玩家）：**
- **命运值**：初始 50，影响关键事件触发概率和隐藏结局
- **忠义值**：初始值因武将而异，影响人际关系事件和背叛/忠诚分支

**属性变化规则：**
- 每个事件选择改变 1-3 个四维属性（明确显示变化值）
- 隐藏属性随选择暗中变化（不显示具体数值）
- 属性上限 100，下限 0

## 3. 阶段系统

游戏分为四个线性人生阶段，每个阶段有独立的事件池。

| 阶段     | 主题       | 事件数   | 说明                                         |
|----------|-----------|----------|----------------------------------------------|
| 少年期   | 成长与学习 | 5-6 个   | 拜师、习武、读书、初遇名将，奠定属性基调      |
| 崛起期   | 投奔与建功 | 6-8 个   | 选择阵营、初战、立功、结交盟友/树敌           |
| 争霸期   | 乱世抉择   | 6-8 个   | 大战役、政治阴谋、生死抉择、背叛或忠诚       |
| 终局期   | 命运归宿   | 3-4 个   | 最终决战/谈判/退隐，根据属性总和触发结局      |

**阶段内机制：**
- 每个阶段从事件池中随机抽取固定数量的事件（保证复玩不同体验）
- 阶段间自动过渡，有简短的过渡叙述文字
- 每个阶段内的事件顺序随机

## 4. 事件系统

### 4.1 事件结构

每个事件包含：
- **标题**：事件名称
- **描述**：100-200 字的场景描述
- **前置条件（可选）**：属性阈值条件，满足才会进入事件池
- **2-3 个选择**，每个选择包含：
  - 选择文本
  - 四维属性变化效果（显示给玩家）
  - 隐藏属性变化效果（不显示）
  - 选择结果叙述文本

### 4.2 事件 JSON Schema

```json
{
  "id": "string",
  "phase": "少年期 | 崛起期 | 争霸期 | 终局期",
  "title": "string",
  "description": "string",
  "conditions": {
    "武": { "min": 0, "max": 100 },
    "智": { "min": 0, "max": 100 }
  },
  "choices": [
    {
      "text": "string",
      "effects": { "武": 0, "智": 0, "德": 0, "魅": 0 },
      "hidden_effects": { "命运": 0, "忠义": 0 },
      "result": "string"
    }
  ]
}
```

### 4.3 隐藏选项机制

部分事件有隐藏的第 3 个选项，只在特定隐藏属性满足条件时出现。隐藏选项通过 `hidden_conditions` 字段控制：

```json
{
  "text": "舍身取义",
  "hidden_conditions": { "忠义": { "min": 80 } },
  "effects": { "武": -5, "德": +10 },
  "hidden_effects": { "命运": +10, "忠义": +10 },
  "result": "..."
}
```

### 4.4 事件数量规划

- 少年期事件池：8-10 个（每局随机抽 5-6 个）
- 崛起期事件池：10-12 个（每局随机抽 6-8 个）
- 争霸期事件池：10-12 个（每局随机抽 6-8 个）
- 终局期事件池：6-8 个（每局随机抽 3-4 个）
- **总计：34-42 个事件**

## 5. 结局系统

### 5.1 结局列表（7 个 + 1 个默认）

| 结局         | 触发条件                           | 描述                                       |
|-------------|------------------------------------|--------------------------------------------|
| 一统天下     | 武≥80, 智≥80, 魅≥75               | 文武兼备，众望所归，终结乱世建立新朝         |
| 忠臣良将     | 武≥70, 德≥85, 忠义≥80              | 一生忠义，名留青史，成为后世武将楷模         |
| 幕后军师     | 智≥90, 魅≥70, 武<60               | 运筹帷幄，辅佐明主，以谋略定天下            |
| 乱世枭雄     | 武≥75, 智≥75, 德<40               | 不择手段成就霸业，功过任后人评说             |
| 归隐山林     | 德≥80, 命运≥70, 无极端属性值       | 看透乱世纷争，隐于山水之间，著书立说         |
| 战死沙场     | 武≥85, 智<50, 忠义≥60              | 勇猛无匹但缺少谋略，壮烈牺牲               |
| 众叛亲离     | 德<30, 忠义<30                     | 背信弃义终遭反噬，被昔日盟友联手围剿         |
| 平凡一生     | 不满足以上任何条件（默认）           | 在乱世中艰难求存，最终老死乡野               |

### 5.2 结局判定逻辑

- 终局期结束后，按表格从上到下顺序匹配
- 返回第一个满足所有条件的结局
- 若无匹配，触发"平凡一生"默认结局
- "无极端属性值"定义：所有四维属性在 30-85 之间

### 5.3 结局展示

每个结局画面包含：
- 结局名称（大字标题）
- 结局故事（200-300 字）
- 最终四维属性数值展示
- 人生回顾（列出关键事件选择）
- "再来一局"和"结局图鉴"按钮

### 5.4 结局图鉴

- 单独的图鉴页面，展示所有 8 个结局
- 已解锁的结局显示名称、缩略描述
- 未解锁的结局显示"???"剪影
- 解锁进度保存在 localStorage，跨局保留

## 6. UI 设计

### 6.1 视觉风格

- **色调：** 暗色古风（深蓝底 #1a1a2e + 金色文字 #c4a24e + 暖色调 #e0c097）
- **字体：** 系统衬线字体为主，标题用 serif
- **风格：** 文字 + 简单插画，以简约的图标和边框装饰营造古风氛围
- **适配：** 移动优先设计，竖屏为主，兼容 PC 浏览器

### 6.2 核心界面

**5 个主要画面：**

1. **标题画面**：游戏标题 + "开始征途"按钮 + "结局图鉴 (X/8)"按钮
2. **选将画面**：3×2 网格展示 6 位武将卡片，显示四维属性和定位标签，点击选择
3. **事件画面**：顶部状态栏（当前阶段/进度）+ 四维属性条 + 事件描述文本 + 2-3 个选择按钮（每个按钮显示属性变化预览）
4. **选择结果画面**：结果叙述文本 + 属性变化气泡（绿色增加/红色减少）+ "继续"按钮
5. **结局画面**：结局名称 + 故事文本 + 最终属性 + "再来一局"/"结局图鉴"按钮

### 6.3 交互流程

```
标题画面 → 选将画面 → [事件画面 → 选择结果画面] × N → 结局画面
                                                          ↓
                                                     结局图鉴 / 重玩
```

## 7. 技术架构

### 7.1 文件结构

```
G:/TestGame/
├── index.html              # 入口页面（单页应用）
├── css/
│   └── style.css           # 全局样式（古风暗色主题）
├── js/
│   ├── game.js             # 游戏引擎（状态管理、事件循环、结局判定）
│   ├── ui.js               # UI 渲染（画面切换、动画、属性展示）
│   └── data-loader.js      # JSON 数据加载器
├── data/
│   ├── characters.json     # 武将数据
│   ├── events-youth.json   # 少年期事件池
│   ├── events-rise.json    # 崛起期事件池
│   ├── events-war.json     # 争霸期事件池
│   ├── events-final.json   # 终局期事件池
│   └── endings.json        # 结局定义与触发条件
└── img/                    # 插画资源（武将头像、结局插画，后续添加）
```

### 7.2 游戏状态对象

```json
{
  "character": "关羽",
  "phase": 1,
  "eventIndex": 0,
  "attrs": { "武": 95, "智": 70, "德": 90, "魅": 80 },
  "hidden": { "命运": 50, "忠义": 70 },
  "history": ["youth_01:choice_a", "youth_03:choice_c"],
  "unlockedEndings": ["忠臣良将"]
}
```

### 7.3 数据持久化

- **游戏进度：** 保存到 `localStorage`（key: `sgwjl_save`），支持刷新页面续玩
- **结局解锁记录：** 独立保存到 `localStorage`（key: `sgwjl_endings`），跨局保留
- 每次做出选择后自动保存

### 7.4 核心引擎逻辑

1. **初始化：** 加载所有 JSON 数据 → 检查 localStorage 是否有存档
2. **事件循环：** 当前阶段事件池 → 过滤满足条件的事件 → 随机抽取 → 展示 → 等待选择
3. **选择处理：** 应用属性变化 → 记录历史 → 自动保存 → 展示结果
4. **阶段推进：** 当前阶段事件完成 → 过渡动画 → 进入下一阶段
5. **结局判定：** 终局期结束 → 按优先级遍历结局条件 → 匹配第一个 → 展示结局
