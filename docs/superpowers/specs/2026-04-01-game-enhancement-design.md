# 三国武将录 — 游戏增强设计文档

> 日期：2026-04-01
> 状态：已批准

## 1. 设计目标

为「三国武将录」文字冒险游戏引入三个紧密关联的系统：

1. **夭折系统** — 玩家中途可能因属性过低或极端选择导致游戏提前结束
2. **角色专属事件** — 每个角色每阶段有 1 个必触发的独特事件，形成各自的故事线
3. **事件链 + 标记系统** — 前期选择通过 flag 影响后期事件，产生因果叙事

## 2. 系统交互关系

```
角色专属事件 ──────────┐
  (每阶段1个必触发)     │
                       ├──→ 事件链/Flag系统 ──→ 跨阶段因果叙事
通用随机事件 ──────────┤        ↑
  (现有+扩展)          │        │
                       │   set_flags / requires_flags
                       │
                       └──→ 夭折系统
                              ├── 属性危机检测（阶段末）
                              ├── 危机事件抉择
                              └── 即死选项（特定事件）
```

## 3. 夭折系统

### 3.1 触发机制

采用**两者结合**方案：属性阈值触发 + 关键事件即死选项。

#### 属性危机触发

每个阶段**所有事件处理完毕后**，检测玩家属性是否进入危险区：

| 检测属性 | 危险阈值 | 对应危机类型 |
|---------|---------|------------|
| 命运 | < 15 | 病殁危机 |
| 德 < 12 且 魅 < 20 | 复合判定 | 遇刺危机 |
| 武 | < 12（仅争霸期/终局期） | 阵亡危机 |
| 忠义 < 12 且 德 < 20 | 复合判定 | 问斩危机 |

触发后，在进入下一阶段前插入一个**危机事件**。

#### 危机事件

危机事件提供 2-3 个选择，部分选择需要属性判定：
- 判定成功 → 化解危机，继续游戏（可能获得少量属性恢复）
- 判定失败 → 进入夭折结局

示例（病殁危机）：
```json
{
  "id": "crisis_illness",
  "title": "病入膏肓",
  "description": "连日操劳，你感到身体每况愈下。一日晨起，竟咳出血来……",
  "crisis_type": "illness",
  "choices": [
    {
      "text": "寻访名医全力医治",
      "crisis_check": { "命运": { "min": 10 } },
      "success_result": "幸得名医妙手回春，你从鬼门关前捡回一命。",
      "success_effects": { "命运": 10 },
      "fail_result": "名医也回天乏术，你的生命如风中残烛……",
      "death_ending": "death_illness"
    },
    {
      "text": "强撑病体继续征程",
      "crisis_check": { "武": { "min": 25 } },
      "success_result": "凭着一股意志力，你挺过了最危险的时期。",
      "success_effects": { "武": -5, "命运": 5 },
      "fail_result": "体力终于耗尽，你倒在了行军途中……",
      "death_ending": "death_illness"
    },
    {
      "text": "将后事托付部下，安心养病",
      "result": "你放下一切安心休养，数月后奇迹般康复。",
      "effects": { "武": -3, "智": -2, "命运": 15 }
    }
  ]
}
```

第三个选项是"安全出口"——代价是属性下降，但保证存活。

#### 即死选项

少量通用事件中的极端选项可直接触发夭折判定。这些选项必须有**明确的危险提示**文字（如加前缀「⚠️ 危险：」），让玩家知道风险。

即死选项的数据结构：
```json
{
  "text": "⚠️ 危险：孤身闯入敌营刺杀",
  "effects": { "武": 8, "魅": 5 },
  "crisis_trigger": true,
  "crisis_check": { "武": { "min": 40 }, "智": { "min": 30 } },
  "death_ending": "death_battle",
  "success_result": "你成功刺杀敌将，一战成名！",
  "fail_result": "你被敌军识破，陷入重围……"
}
```

### 3.2 夭折结局（4 个，计入图鉴）

| ID | 名称 | 图标 | 触发条件 | 故事概要 |
|----|------|------|---------|---------|
| `death_illness` | 病殁沙场 | 💀 | 命运耗尽，危机事件未能化解 | 英雄未竟之志，终被病魔击倒。一代枭雄，竟不能寿终正寝。 |
| `death_assassinated` | 遇刺身亡 | 🗡️ | 德/魅过低，树敌过多 | 你树敌太多，终有一日暗箭难防。在一个月黑风高之夜，刺客来访…… |
| `death_battle` | 马革裹尸 | ⚔️ | 武过低于战场阶段 | 战场之上容不得半分犹豫，你的武艺终究不足以在这修罗场中立足。 |
| `death_executed` | 身首异处 | ⛓️ | 忠义/德极低，被视为叛贼 | 你的所作所为终于激起众怒，各方势力联名上表，将你定为乱臣贼子…… |

结局图鉴从 13 → 17 个。夭折结局在图鉴中以特殊颜色标识（暗红色边框）。

### 3.3 夭折判定流程

```
阶段所有事件完成
        │
        ▼
  检测属性是否进入危险区 ──── 否 ──→ 正常进入下一阶段
        │ 是
        ▼
  插入对应的危机事件
        │
  玩家做出选择
        │
        ├── 安全选项（有代价） ──→ 继续游戏
        ├── 判定选项 + 成功 ──→ 继续游戏
        └── 判定选项 + 失败 ──→ 夭折结局
```

注意：少年期（第一阶段）不触发夭折检测，从崛起期开始检测。给玩家至少一个阶段的缓冲。

## 4. 角色专属事件系统

### 4.1 机制

- 每个角色在每个阶段有 **1 个必触发的专属事件**
- 专属事件在该阶段的**第一个事件**位置触发（先专属后随机）
- 随机事件的抽取数相应减 1（保持每阶段总事件数不变）
  - 少年期：1 专属 + 4 随机 = 5
  - 崛起期：1 专属 + 5 随机 = 6
  - 争霸期：1 专属 + 5 随机 = 6
  - 终局期：1 专属 + 2 随机 = 3

### 4.4 代码变更

`game.js` 中 `_pickPhaseEvents()` 修改逻辑：

```
1. 从 events-character.json 中取出当前角色+当前阶段的专属事件
2. 从通用事件池中 filter + shuffle 抽取 (pickCount - 1) 个事件
3. 将专属事件放在数组首位
4. 返回 [专属事件, ...随机事件]
```

## 5. 事件链 + 标记（Flag）系统

### 5.1 机制

#### 数据结构扩展

**选项新增字段：**
```json
{
  "text": "救下孤儿",
  "effects": { "德": 5 },
  "set_flags": ["saved_orphan"]
}
```

**事件 conditions 新增字段：**
```json
{
  "requires_flags": ["saved_orphan"],
  "excludes_flags": ["abandoned_orphan"]
}
```

#### 状态扩展

`state` 中新增 `flags` 数组：
```json
{
  "characterId": "farmer",
  "attrs": { ... },
  "hidden": { ... },
  "flags": ["saved_orphan", "spy_clue"],
  "history": [ ... ]
}
```

#### 判定逻辑

在 `_checkConditions()` 中增加 flag 检查：
- `requires_flags`：state.flags 必须包含列表中的**所有** flag
- `excludes_flags`：state.flags 必须**不包含**列表中的任何 flag

### 5.2 通用事件链设计（4 条）

#### 链1：孤儿线

| 阶段 | 事件 | 触发条件 | 设置标记 |
|------|------|---------|---------|
| 少年 | 路遇孤儿 — 战乱中遇到一个走失的孩子 | 无 | 选择救下 → `saved_orphan` |
| 崛起 | 少年来投 — 当年救下的孩子已长大，前来投奔 | `requires: saved_orphan` | 收留 → `orphan_joined` |
| 争霸 | 绝境救主 — 危急时刻，忠心部下舍命相救 | `requires: orphan_joined` | — |

#### 链2：间谍暗线

| 阶段 | 事件 | 触发条件 | 设置标记 |
|------|------|---------|---------|
| 崛起 | 蛛丝马迹 — 发现军中有人暗通敌营 | 无 | 选择追查 → `spy_clue` |
| 争霸 | 顺藤摸瓜 — 间谍线索逐渐明朗 | `requires: spy_clue` | 成功追踪 → `spy_traced` |
| 终局 | 真相大白 — 内奸身份揭露，竟是身边亲信 | `requires: spy_traced` | — |

#### 链3：神医线

| 阶段 | 事件 | 触发条件 | 设置标记 |
|------|------|---------|---------|
| 少年 | 救助伤者 — 路遇受伤的行脚医者 | 无 | 选择救助 → `saved_doctor` |
| 争霸 | 妙手回春 — 关键战役中神医出手救治伤兵 | `requires: saved_doctor` | — |

此链与**夭折系统联动**：若触发病殁危机且拥有 `saved_doctor` flag，危机事件中额外出现一个高成功率选项。

#### 链4：宝物线

| 阶段 | 事件 | 触发条件 | 设置标记 |
|------|------|---------|---------|
| 崛起 | 古籍残卷 — 偶然获得一本残破的兵法古籍 | 无 | 选择研读 → `ancient_scroll` |
| 争霸 | 破解奥秘 — 古籍中的奇策在关键战役中发挥作用 | `requires: ancient_scroll` | — |

### 5.3 角色专属事件链（6 条，每角色 1 条）

利用专属事件之间的 flag 串联，形成角色专属因果叙事：

| 角色 | 前置事件 | 前置选择 | 后续触发 | 效果 |
|------|---------|---------|---------|------|
| 农夫 | 少年「离乡赴难」 | 带领乡亲出走 → `farmer_led_villagers` | 崛起「组建义军」中乡亲成为骨干 | 额外魅+5 |
| 书生 | 少年「师门求学」 | 追随名师 → `scholar_followed_master` | 争霸「舌战群雄」中获得师门传承加成 | 额外智+5 |
| 士卒 | 少年「军中历练」 | 团结战友 → `soldier_united_comrades` | 崛起「阵前封将」中战友助阵 | 额外武+5 |
| 游侠 | 少年「行侠仗义」 | 救下某人 → `wanderer_saved_stranger` | 争霸「刺杀任务」中此人提供情报 | 额外智+3, 命运+5 |
| 商贾 | 少年「商路奇遇」 | 仗义疏财 → `merchant_generous` | 崛起「战时生意」中获得商路人脉 | 额外魅+3, 命运+5 |
| 匠人 | 少年「器械之道」 | 精研防具 → `craftsman_defense_focus` | 争霸「奇门利器」中解锁防御型发明 | 额外德+5, 命运+3 |

### 5.4 设计原则

- 每条链跨 2-3 个阶段，不超过 3 个事件
- 链条中断不影响游戏完整性 — 只是少看到后续剧情
- flag 名称使用 snake_case，语义清晰
- 通用链和角色专属链互不冲突

## 6. 游戏流程变化

### Before

```
选角色 → 5+6+6+3=20 个随机事件 → 13种结局
```

### After

```
选角色 → 阶段循环 {
  1. 角色专属事件（1个，必触发）
  2. 随机事件（从通用池+链条事件中抽取）
     - 少年: 4个  崛起: 5个  争霸: 5个  终局: 2个
  3. 阶段末危机检测（少年期除外）
     ├── 安全 → 下一阶段
     └── 危险 → 危机事件
         ├── 化解 → 下一阶段
         └── 失败 → 夭折结局
} → 正常结局（13个）或 夭折结局（4个）
```

每局事件数：4 专属 + 16 随机 + 0~2 危机 = **20~22 个事件**
