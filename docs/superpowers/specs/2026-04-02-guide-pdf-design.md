# 三国武将录 · 攻略PDF 设计文档

**日期：** 2026-04-02  
**项目：** 三国武将录（San Guo Wu Jiang Lu）  
**目标：** 为游戏生成一份图文并茂的精华版攻略PDF

---

## 一、目标与受众

### 受众
- **新手玩家**：通过前三章快速上手，了解系统机制和武将特点
- **成就党 / 全结局收集者**：通过后三章查询精确的结局解锁条件和路线建议

### 成功标准
- 新手读完第一章能理解属性系统和游戏流程
- 高玩能通过第四、五章查到任意结局的精确解锁条件
- PDF 可正常显示中文，排版清晰美观
- 数据与游戏 JSON 文件 100% 一致（脚本自动生成）

---

## 二、内容结构（约20页）

```
封面
第一章：游戏入门（系统说明、属性解读、游戏流程）
第二章：六位武将详解（属性表 + 路线推荐 + 适合结局）
第三章：危机事件必读（4种危机 + 应对策略）
第四章：结局图鉴（17种结局的解锁条件速查表）
第五章：全结局路线指南（按结局分类的关键属性门槛）
附录：标记系统说明
```

### 双层设计原则
- 新手读前三章，高玩读后三章
- 精华版原则：保留部分游戏体验，不逐一列出每个选项的属性影响

## 三、技术方案

### 工具链
- **生成脚本：** `guide/generate_guide.py`（Python 3，无第三方依赖）
- **PDF转换：** Pandoc + XeLaTeX 引擎（支持中文）
- **中文字体：** SimSun（宋体）或系统可用中文字体

### 文件结构
```
guide/
├── generate_guide.py         # 主脚本：读取JSON → 输出 guide.md
├── templates/
│   └── manual_content.py     # 手写内容模块（路线推荐、危机技巧等）
├── guide.md                  # 自动生成的完整攻略 Markdown
└── guide.pdf                 # Pandoc 转换后的最终产物
```

### 生成流程
```bash
# 第一步：生成 Markdown
python guide/generate_guide.py

# 第二步：转换为 PDF
pandoc guide/guide.md -o guide/guide.pdf \
    --pdf-engine=xelatex \
    -V mainfont="SimSun" \
    -V geometry:margin=2cm \
    --toc \
    --toc-depth=2
```

### 数据来源分工

| 章节 | 数据来源 | 生成方式 |
|------|---------|---------|
| 封面 + 第一章 | 手写 | manual_content.py 中的字符串 |
| 第二章 武将详解 | `data/characters.json` | 脚本自动生成属性表格 + 手写路线建议 |
| 第三章 危机攻略 | `data/events-crisis.json` | 脚本提取危机信息 + 手写策略提示 |
| 第四章 结局图鉴 | `data/endings.json` | 脚本自动生成17行速查表 |
| 第五章 路线指南 | `data/endings.json` + 手写 | 手写推荐路线，脚本提供属性门槛数据 |
| 附录 标记系统 | `data/events-*.json` | 脚本扫描 set_flags + 手写说明 |

## 四、各章内容设计

### 第一章：游戏入门

内容：
- 游戏概述（4阶段、17结局、危机系统）
- 六大属性说明：武/智/德/魅（可见）、命运/忠义/魏/蜀/吴（隐藏）
- 属性范围：所有属性限制在 [0, 100]
- 结局判定顺序说明（死亡结局优先，正常结局按序匹配）

### 第二章：六位武将详解

每位武将一张卡片，格式：

```
## [图标] [武将名]「[称号]」

| 属性 | 初始值 | 特点 |
|------|--------|------|
| 武   |  xx   | 高/中/低 |
| 智   |  xx   | 高/中/低 |
| 德   |  xx   | 高/中/低 |
| 魅   |  xx   | 高/中/低 |

**隐藏属性：** 命运xx · 忠义xx

**新手推荐指数：** ⭐（1-5星）

**推荐结局：** [结局A] / [结局B]

**路线提示：** [手写的简短策略建议]
```

难度评级标准：
- ⭐⭐⭐⭐⭐ 商贾之子（魅力流，路线独特）
- ⭐⭐⭐ 农夫之子（均衡型，容错率高，新手首选）

### 第三章：危机事件必读

每种危机的格式：

```
### [emoji] [危机名称]
**触发时机：** 阶段结束时（少年期免疫）
**安全出口：** [选项名称]
  → 代价：[属性变化]，但保证存活
**判定选项：** [选项名称]（需 [属性]≥[阈值]）
**建议：** [手写的一句话策略]
```

四种危机：病入膏肓 / 月黑杀机 / 兵临绝境 / 秋后问斩

特别说明：`saved_doctor` 标记为病殁危机提供额外高成功率选项。

### 第四章：结局图鉴

完整速查表，共17行，全部结局必须列出（脚本从 endings.json 自动生成，不得省略）。示例格式：

```
| 结局       | 类型 | 关键条件                          | 难度   |
|------------|------|-----------------------------------|--------|
| 一统天下   | 正常 | 武≥85, 智≥85, 魅≥80              | ★★★★★ |
| 幕后军师   | 正常 | 智≥88, 魅≥68, 武≤55              | ★★★★  |
| 归隐山林   | 正常 | 德≥75, 命运≥62, 无极端属性        | ★★★   |
| ...（其余11种正常结局同格式）...           |        |
| 平凡一生   | 正常 | 兜底结局（无条件匹配）            | ★      |
| 病殁沙场   | 死亡 | 触发病殁危机且判定失败            | —      |
| 遇刺身亡   | 死亡 | 触发暗杀危机且判定失败            | —      |
| 马革裹尸   | 死亡 | 触发战场危机且判定失败            | —      |
| 身首异处   | 死亡 | 触发处斩危机且判定失败            | —      |
```

死亡结局4种单独列出，标注触发方式而非属性条件。

### 第五章：全结局路线指南

按结局分组，每个结局给出：
- 推荐起始武将（哪个武将初始属性最接近目标）
- 核心属性目标（需达到的关键门槛）
- 阵营倾向提示（魏/蜀/吴隐藏属性影响）
- 需要避免的属性上限（如 `武≤55` 的幕后军师路线）

### 附录：标记系统说明

已知关键标记速查表：

| 标记名 | 触发条件 | 解锁效果 |
|--------|---------|---------|
| farmer_led_villagers | 农夫之子·少年期·带乡亲撤离 | 崛起期解锁「组建义军核心」选项（魅+10） |
| joined_coalition | 少年期·冷静分析战局 | 后续联盟相关事件解锁 |
| saved_doctor | 特定事件中救治神医 | 病殁危机新增高成功率选项 |

## 五、风格规范

- **语言风格：** 简体中文，信息密度高，保留少量古风词汇呼应游戏气质
- **Markdown 规范：**
  - 标题层级最多三级（#/##/###）
  - 表格对齐，列宽均匀
  - 关键数值用**加粗**标出
  - 危机和死亡内容用 `⚠️` 标注
- **PDF排版：**
  - 页边距 2cm
  - 自动生成目录（`--toc`）
  - 中文宋体，正文12pt

---

## 六、实现约束

- Python 脚本仅使用标准库（`json`, `os`, `pathlib`），无需 pip install
- 脚本运行目录为项目根目录（`G:/TestGame/`）
- 手写内容集中在 `manual_content.py` 中，便于未来维护
- 脚本需处理 `endings.json` 中 `conditions` 为空对象 `{}` 的兜底结局（平凡一生）
- 结局条件中的 `no_extreme` 需用人类可读语言描述（"任意可见属性不低于30且不超过85"）
